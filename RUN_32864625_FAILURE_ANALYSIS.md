- 治疗组 "D-Exp"：250mg 有 0 人，300mg 有 0 人，350mg 有 10 人

### Round 3 代码分析

**数据加载**（✅ 路径正确）：
```python
df = pd.read_csv('public/workdir/data_clinical_patient.csv')
```

**透视表构建**（❌ 核心问题）：
```python
dose_counts = df.groupby(['TREATMENT_ARM', 'ALPELISIB_DOSE']).size().unstack(fill_value=0)
dose_counts.columns.name = 'ALPELISIB_DOSE'
```

### 疑似问题点

1. **列数据类型错误**：
   - `ALPELISIB_DOSE` 列可能是字符串类型（如 `'250'`、`'300'`、`'350'`）
   - 但测试断言使用的是整数（`250`、`300`、`350`）
   - 导致 `dose_counts.loc["A", 250]` 找不到列 `250`

2. **列名不存在**：
   - 数据中可能没有剂量为 250、300、350 的记录
   - 或者剂量值的格式不同（如 `'250 mg'`、`'250.0'`）

3. **列名转换缺失**：
   - AI 应该将 `ALPELISIB_DOSE` 列转换为整数类型
   - 或者在 `unstack()` 后将列名转换为整数

4. **数据筛选缺失**：
   - 可能需要先筛选出有效的剂量记录
   - 或者排除 NaN 值

### 正确的代码应该是

```python
import pandas as pd

df = pd.read_csv('public/workdir/data_clinical_patient.csv')

# 确保 ALPELISIB_DOSE 是数值类型
df['ALPELISIB_DOSE'] = pd.to_numeric(df['ALPELISIB_DOSE'], errors='coerce')

# 构建透视表
dose_counts = df.groupby(['TREATMENT_ARM', 'ALPELISIB_DOSE']).size().unstack(fill_value=0)

# 确保列是整数类型
dose_counts.columns = dose_counts.columns.astype(int)
dose_counts.columns.name = 'ALPELISIB_DOSE'

dose_counts.to_csv('outputs/dose_counts.csv')
```

### 为什么 AI 3 轮都没修正？

错误反馈只是：
```
KeyError: 250
```

这个反馈**没有明确告诉 AI**：
- 错误发生在哪一行代码
- `dose_counts` 的实际列名是什么（如 `['250', '300', '350']` 还是其他）
- 是数据类型问题还是列不存在问题

如果反馈能包含：
```
KeyError: 250
Available columns: ['250', '300', '350'] (type: str)
Expected columns: [250, 300, 350] (type: int)
```

AI 就能立即理解是数据类型不匹配的问题。
