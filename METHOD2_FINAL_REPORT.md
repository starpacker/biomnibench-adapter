5. **状态持久化** - 每个子任务有独立的状态文件

### 执行流程
```
for 每个子任务 (1-118):
  for 尝试 (1-3):
    1. 收集前面已完成任务的输出和代码
    2. 调用AI生成当前子任务的代码
    3. 执行代码，生成输出
    4. Judge立即验证
    5. 通过 → 保存，进入下一个子任务
       失败 → 记录错误，重试当前子任务
```
